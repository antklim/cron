Данная программа ( далее "cron" ) предназначена для запуска 
произвольного .hta приложения и перезапуска этого приложения 
при его закрытии.

Требования к системе:
1. Наличие переменной окружения TERMPATH, в которой указывается 
   директорий файлов настройки ( с закрывающим слешем )
   В случае отсутствия таковой переменной окружения рабочим
   каталогом будет считаться 
   c:\Documents and Settings\< USER >\Application Data\Term
   (если каталог Term не найден в c:\Documents and Settings\< USER >\Application Data
   то он будет автоматически создан)
   В случае наличия переменной окружения TERMPATH и указания 
   в ней пути к несуществующему каталогу - приложение не запустится
   и информация об ошибке будет записана в cron.log в тот же каталог,
   где находится cron.exe

2. В директории, указанной в переменной окружения TERMPATH должны
   существовать два файла: cron.app , cron.log .
   В первой строке файла cron.app должен быть указать путь к .hta приложению
   Файл cron.log используется для логирования событий "cron"
   
Рабочий цикл:
1. "cron" стартует и проверяет наличие переменной окружения
   и существование фалов cron.app , cron.log ( без нахождения файла cron.app 
   приложение не запустится и "cron" завершит свою работу )

2. Старт .hta приложения, указанного в файле cron.app

3. Повторный запуск .hta приложения как только обноружено,
   что оно закрыто

Перезапуск приложения:
Для перезапуска приложения необходимо в каталоге,
указанном в переменной окружения, создать файл cron.rst. Как только
будет обнаружен этот файл произойдет перезапуск .hta приложения. 
Если содержимое файла cron.rst пусто, то приложение будет перезапущено
через 1000 мсек, иначе перезапуск произойдет через указанное в файле 
количество мсек. В случае невозможности получить из содержимого файла 
целое число перезапуск также произойдет через 1000 мсек.
Файл cron.rst будет автоматически удален. В случае, если удаление 
файла невозможно по каким-то причинам запись об этом появится в 
cron.log файле и .hta приложение до тех пор пока не будет удален 
cron.rst файл, либо пока не будет найден cron.stp файл
   
Остановка приложения:
Для остановки работы приложения необходимо в каталоге, 
указанном в переменной окружения, создать файл cron.stp. Как только
будет обнаружен этот файл произойдет завершение работы .hta 
приложения и "cron".
При обнаружении stp файла в каталоге, указанном в переменной 
окружения, во время старта "cronа" он не запустится.
Файл cron.stp имеет больший приоритет, чем cron.rst, то есть при одновременном
наличии этих двух файлов в каталоге, указанном в переменной 
окружения, произойдет остановка приложения

Отложенный старт:
По умолчанию .hta приложение стартует через 1000 мсек после старта "cron".
Если на момент запуска "cron"  каталоге, указанном в переменной окружения,
обнаружен файл cron.rst, то запуск .hta приложения будет отложен на 
количество мсек, указанных в файле. Если файл пустой, либо содержимое
невозможно привести к целому числу - запуск происходит по умолчанию.

===============================================================================
*                       ИЗМЕНЕНИЯ В ВЕРСИИ CRON_V2                            *
===============================================================================
1. При запуске проверяется наличие cron.stp файла в том же каталоге, что и 
   cron.exe. В случае обнаружения такого файла "cron" будет остановлен.  
2. "cron" определяет рабочий каталог из TERMPATH и не останавливается, если не 
   удается обнаружить указанный каталог до тех пор, пока он не появится, либо
   не будет создан cron.stp файла в том же каталоге, что и cron.exe.
3. При запуске "crona" закрывается окно консоли, однако процесс продолжает 
   работать.

===============================================================================
*                       ИЗМЕНЕНИЯ В ВЕРСИИ CRON_V3                            *
===============================================================================
1. При обнаружении cron.stp *.hta приложение закрывается, но "cron" продолжает 
   работать. Как только cron.stp файл удаляется cron вновь запускает *.hta
   приложение.
2. "Мягкое" завершение работы *.hta приложения. Вызов TermainteProcess заменен 
   отправкой сообщения о закрытии окна *.hta приложению.
